end state; they are not claims that a feature is available. This page is updated
in the same pull request whenever implementation status changes.

The current operator plan starts poolers only in explicit
`bootstrap-unavailable` catalog mode. This proves a credential-free process can
remain alive and observable before catalog transport exists; phase
`not_configured` and readiness reason `catalog_not_configured` remain
fail-closed, and no connection attempt is made. It is not evidence of a usable
pooler or database endpoint.

No development database cluster can be installed from the current source. The
admission Kustomization installs the CRD, manager, self-managed serving
certificate, and fail-closed webhook configurations, while a separate
certificate-free overlay remains available for controller debugging. Automatic
CA rotation is not implemented. The operator does not create PostgreSQL Pods
or PVCs, the pooler accepts no wire session and has no connection pool, and
supporting Services are not usable application endpoints. No runtime
correctness, availability or performance guarantee is claimed until its
implementation and required tests are merged and listed here.
